# Screenshot Plan

Screenshots are planned but not committed yet. Add image files only after the
commands are reproducible on a clean workstation and the resulting screenshots
match the lesson text.

| Group | Directory | Purpose |
| --- | --- | --- |
| Setup | [setup/](setup/) | Windows/WSL, install, validation, and first case creation. |
| Workflow | [workflow/](workflow/) | CLI and case-builder workflow states. |
| MCP | [mcp/](mcp/) | MCP registration, tool listing, and tool output examples. |

## Capture Rules

- Use real terminal or MCP-host states from `mkultra_course`.
- Redact hostnames, private paths outside the repo, tokens, keys, emails,
  browser profiles, and private notes.
- Keep screenshots visually boring and exact: command, output, source ID, and
  relevant path are more important than decoration.
- Do not use generated mock screenshots for evidence workflows.
- Reference screenshot filenames in lesson text only after the files exist.
